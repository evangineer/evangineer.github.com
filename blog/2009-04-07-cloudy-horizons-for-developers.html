Cloudy Horizons for Developers

The following is in part a response to Directgov Innovate’s Prototyping in the Cloud blog post & something of a summary of my own views on the cloud as a web developer.

Cloud hosting as described by the aforementioned post, is what’s known as Infrastructure as a Service. The most well known available services along these lines are Amazon Web Services like EC2 and S3. Another service along those lines is Mosso also known as the Rackspace Cloud. The open source equivalent of AWS is Eucalyptus which is being incorporated into the Ubuntu Server release. Having looked at the APIs for the new Cloud offering from Sun, it fits into this category of cloud hosting.

Speaking as a web developer, I’ve gotten tired of wrestling with system administration issues when deploying/installing web applications. Anything that helps ease deployment removes hassle from the process & allows me to spend my time on more useful activities. There are developer oriented deployment tools like Capistrano and Fabric that can be helpful in this regards.

In the context of cloud computing, a solution for taking the system administration hassle out of deployment is what’s called Platform as a Service. The most notable examples of PaaS from my perspective are Google App Engine and Heroku. The reasons they are notable in my view is that Django & other Python web developers can deploy modified versions of their apps onto App Engine and Rails & other Ruby web developers can deploy their apps onto Heroku. Force.com from salesforce.com is also in the PaaS category.

An emergent trend that I’ve found interesting regarding PaaS is the rise of Server-Side Javascript in a break from established server-side dynamic languages like PHP, Perl, Python and Ruby. You see this in Appjet, 10gen and Reasonably Smart Platform. Whereas Appjet is exclusively Javascript based, 10gen and Reasonably Smart are in various stages of incorporating support for Python and Ruby amongst others.

10gen’s application server Babble is written in Java, with the Javascript support courtesy of Rhino. Support for Python and Ruby is being integrated via Jython and JRuby respectively. Reasonably Smart Platform is written in Perl, with Javascript via Spidermonkey. The intention is eventually to harness the IronMonkey project to integrate Python and Ruby via IronPython and IronRuby.

10gen and Reasonably Smart have announced major changes this year. 10gen is shifting emphasis to its cloud database MongoDB based on demand for related support & services and has changed the open source license on Babble to the Apache license in order to invite greater engagement from open source developers. Reasonably Smart has been acquired by the major hosting provider Joyent who have an IaaS offering called Accelerators.

So all that aside, what do I recommend to Directgov Innovate? 1) Use Amazon Web Services for IaaS cloud hosting. Issues have been raised about data protection. Amazon has European AWS farms that you can specifically use to avoid the data leaving the EU. 2) Look at ways of easing deployment for developers, possibly by leveraging deployment tools like Capistrano & Fabric making it easier for developers using major web frameworks like Django & Rails, alternatively incorporating open source PaaS on top of AWS or by implementing multitenant environments like Tyrant. 3) Hire someone to actually work out the details & work with you to implement it!
